TI BASIC Test Suite — Index
============================

46 BASIC test programs covering language features, file I/O, sprites,
graphics, speech, and editor commands. Extracted from TEST_PROGRAMS.txt
in the arduino-cli predecessor repo.

To run a test on a device:
  1. Upload the .bas file via ti_files.py (PC pane → device pane).
     The text file lands as a PROGRAM-type blob.
  2. At the BASIC prompt: OLD "FLASH.testNN"  (or DSK1.testNN if on DSK).
  3. LIST to inspect, RUN to execute.

Multi-part tests: each .bas is the primary / first-part program. The
companion parts (variations, sub-runs, expected-output annotations) all
live in **TEST_PROGRAMS.txt right here in this folder** — search for the
matching section header (e.g. "=== 33. ON BREAK / ON ERROR / ON WARNING ===")
for the full multi-part procedure, the expected output for each run,
and any commentary on TI quirks. The .bas files are the loadable form;
TEST_PROGRAMS.txt is the human-readable spec.

Interactive tests are marked above; they wait for a key press. Use
the BLE keyboard or send a byte over serial to advance them.

Hardware notes:
  - Tests that touch DSK1 need a real .dsk image mounted (test35/36/37/42).
  - test45 wants a BLE gamepad or arrow keys.
  - test46 needs the speech ROM (spchrom.bin baked into the speech
    component).
  - test44 is BASIC-bound at ~few generations/second — slow but visible.
